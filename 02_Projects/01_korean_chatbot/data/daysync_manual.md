# DaySync 팀 일정 관리 시스템 매뉴얼

## 1. 개요

DaySync는 팀원들의 일정과 활동 선호도를 관리하기 위해 사내에서 자체 개발한 일정 관리 시스템이다. 개발 초기 단계에서는 내부적으로 "프로젝트 새벽별(Project Dawnstar)"이라는 코드네임으로 불렸으며, 정식 출시 이후 DaySync로 이름이 바뀌었다.

## 2. 기본 동작 방식

DaySync는 매주 일요일 자정에 팀원별 주간 일정을 갱신한다. 일정 추천 엔진은 기본적으로 7일 주기로 동작하며, 사용자가 별도로 설정하지 않으면 직전 4주간의 활동 기록을 참고하여 다음 주 일정을 추천한다.

활동 추천 점수는 0.0에서 1.0 사이의 값으로 계산되며, 이 점수를 "선호도 점수(Preference Score)"라고 부른다. 선호도 점수가 0.65 이상인 활동은 "선호 활동"으로 분류되어 추천 우선순위가 높아진다.

## 3. API 사용법

DaySync의 일정 조회 API는 기본적으로 9221번 포트에서 서비스된다. 일정 등록 요청은 `POST /schedule` 엔드포인트로 전송하며, 요청 본문에는 팀원 이름과 활동명, 희망 날짜가 포함되어야 한다.

API 응답이 지연되는 경우, 대부분 일정 추천 엔진이 4주치 기록을 다시 계산하는 과정에서 발생한다. 이 경우 평균 응답 시간은 1.2초에서 2.5초 사이이다.

## 4. 설정 항목 (Configuration)

DaySync 관리자 화면에서 조정할 수 있는 주요 설정 항목은 다음과 같다.

- **추천 주기(recommendation_cycle)**: 기본값 7일. 팀 특성에 따라 3일에서 14일 사이로 조정 가능하다.
- **선호도 임계값(preference_threshold)**: 기본값 0.65. 이 값을 낮추면 더 다양한 활동이 추천되지만, 팀원이 실제로 선호하지 않는 활동이 추천될 위험이 커진다.
- **일정 충돌 허용 모드(conflict_mode)**: "strict"(충돌 시 등록 거부) 또는 "soft"(충돌해도 경고만 표시) 중 선택 가능하다. 기본값은 "strict"이다.

## 5. 일정 충돌 처리

같은 시간대에 두 개 이상의 활동이 겹치는 경우를 "일정 충돌"이라고 한다. DaySync는 일정 충돌이 발생하면 충돌 코드를 부여하는데, 가장 흔하게 보고된 충돌 코드는 SC-114이며, 이는 "동일 요일에 거절(아니) 응답과 수락(응) 응답이 동시에 기록된 경우"를 의미한다.

SC-114는 보통 팀원이 같은 활동에 대해 짧은 시간 내에 응답을 두 번 보냈을 때 발생하며, 이 경우 시스템은 가장 마지막에 들어온 응답을 최종 값으로 채택한다.
