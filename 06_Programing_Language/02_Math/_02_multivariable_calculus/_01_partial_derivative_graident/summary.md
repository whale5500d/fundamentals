# 편미분(Partial Derivative)과 기울기 벡터(Gradient)

## 1.1 배경 (필요성)

- 단변수 함수 f(x)의 미분은 x가 조금 변할 때 f(x)가 얼마나 변하는지를 측정함. 그런데 AI 모델의 손실 함수(loss function)는 수백만 개의 파라미터(가중치)를 변수로 가지는 다변수 함수임. L(w₁, w₂, ..., wₙ)에서 손실을 줄이려면 각 파라미터가 손실에 얼마나 영향을 미치는지 개별적으로 측정해야 함. 여기서 편미분을 사용함.
- gradient는 모든 파라미터에 대한 편미분을 하나의 벡터로 모은 것으로, "어느 방향으로 파라미터를 바꿀 때 손실이 가장 빠르게 증가하는가"를 나타냄. gradient의 반대 방향으로 이동하면 손실이 가장 빠르게 감소하며, 이것이 gradient descent(경사 하강법)의 핵심 원리임.

## 1.2 수학적 정의

편미분(partial derivative)

- 다변수 함수 f(x₁, x₂, ..., xₙ)에서 xᵢ에 대한 편미분은 다른 변수를 모두 상수로 고정하고 xᵢ만 변화시킬 때의 변화율임.

```
∂f/∂xᵢ = lim(h→0) [f(x₁, ..., xᵢ+h, ..., xₙ) - f(x₁, ..., xᵢ, ..., xₙ)] / h
```

수치 미분(numerical differentiation)

- 극한을 직접 계산할 수 없으므로 h를 매우 작은 값으로 근사함. 세 가지 방식이 있음.

```
전방 차분(forward difference):  [f(x+h) - f(x)] / h                → 오차 O(h)
후방 차분(backward difference): [f(x) - f(x-h)] / h                → 오차 O(h)
중앙 차분(central difference):  [f(x+h) - f(x-h)] / (2h)           → 오차 O(h²)
```

- 중앙 차분이 오차가 가장 작으므로 실용 구현에서 사용함.

gradient(기울기 벡터)

- f: ℝⁿ → ℝ에 대해 gradient는 모든 편미분을 모은 n차원 벡터임.

```
∇f(x) = [∂f/∂x₁, ∂f/∂x₂, ..., ∂f/∂xₙ]ᵀ
```

방향 미분(directional derivative)

- 임의의 단위 벡터 u 방향으로의 변화율은 gradient와 u의 내적으로 정의됨.

```
D_u f(x) = ∇f(x) · u
```

- gradient 방향(u = ∇f/‖∇f‖)에서 방향 미분이 최대가 됨. 이것이 gradient가 "가장 가파르게 증가하는 방향"인 이유임.

표 1. gradient 계산 속도 비교 — 순수 Python vs NumPy (단위: ms)
| n (변수 수) | 순수 Python (ms) | NumPy (ms) | speedup |
|---|---|---|---|
| 10 | 0.020 | 0.044 | 0.5x |
| 50 | 0.318 | 0.214 | 1.5x |
| 100 | 1.349 | 0.425 | 3.2x |
| 500 | 31.670 | 2.309 | 13.7x |
| 1,000 | 121.306 | 5.418 | 22.4x |

표 1에서 두 가지 특징을 확인할 수 있음.

- **speedup이 선형대수학 단계들보다 낮음**: 선형대수학의 행렬 연산(matmul, inverse 등)은
  O(n²)~O(n³) 복잡도라 NumPy의 BLAS 루틴 활용 효과가 극적으로 나타남. 반면 gradient
  계산은 변수 수 n만큼 함수를 2번씩 호출하는 구조(총 2n번 함수 호출)로, 함수 호출
  오버헤드가 병목임. NumPy 구현도 내부적으로 동일한 반복 구조를 가지므로 speedup이
  제한적임.

- **소규모(n≤50) 역전**: 선형대수학 단계와 동일한 패턴. NumPy 배열 생성(`copy()`) 고정
  오버헤드가 소규모에서 실제 연산 비용을 초과함.

- **실용적 맥락**: 실제 딥러닝 프레임워크(PyTorch, JAX)는 수치 미분이 아닌 자동 미분
  (automatic differentiation — 계산 그래프를 추적해 정확한 gradient를 계산하는 방법)을
  사용함. 수치 미분은 함수 호출 횟수가 변수 수에 비례해 증가하므로 파라미터가 수백만 개인
  신경망에는 적용 불가능함. 이 한계가 2단계(chain rule)에서 자동 미분의 필요성으로 이어짐.
