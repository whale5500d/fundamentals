# 수 - 심화

## 1교시 - AWS 복습

### EBS

- 기본으로 구성 (아마존 리눅스, 암, 나노, 키 - ktb)
- 구역(Availability Zone)을 확인해야 한다.
- 같은 구역으로 해야 붙는다.
- 볼륨으로 인스턴스 생성
- 생성한 볼륨을 인스턴스에 연결한다.
  - Attach Volume로 2번에다 붙인다. (Device 이름 선택, 장치 하나하나를 파일로 가상화해서 다룸. 그래서 경로로 다룸.)
- disk는 원판이며, AWS에서 storage를 말하기도 함.
- 추가 볼륨을 붙이려면 `sudo -R ec2-user /mnt/data`로 진행

- 참고
  - 재부팅 후에도 마운트를 유지하려면 추가 작업 필요

### EFS

- NAS: 네트워크 스토리지, 주로 회사에서 사용
  - 주로 영상 편집, 데이터를 많이 보관해야 하는 경우 주로 사용
  - 클라우드는 외부로 공유되므로, 프라이빗한 데이터가 있으므로 NAS를 사용하는 회사 존재함.
  - 보안에서 안전하고, 장기적으로 비용이 저렴함.

- EFS는 네트워크 스토리지이긴 하지만, AWS에서 제공하는 퍼블릭 기능
  - 한 쪽에서 수정하면, 다른 쪽에 내용이 업데이트

## 2교시 - AWS 주요 서비스

- 스토리지와 파일 관리
  - EFS
  - S3
    - 대규모 데이터를 안전하게 저장하고 검색할 수 있고 확장까지 가능한 "객체 스토리지 서비스"
    - 아마존을 알게 해준 서비스
    - 구글 드라이브 유료 (Lock-in됨. 지메일 + 제미나이 때문임.)
    - 주로 비정형 데이터일 때 사용 (이미지 등)

- 모니터링과 관리 도구
  - CloudTrail: AWS 계정에서의 API 호출 및 활동을 기록하고 모니터링하는 서비스
  - CloudWatch: AWS 리소스와 애플리케이션에서 발생하는 "로그, 지표, 이벤트"를 "수집, 모니터링"하는 통합 관제 서비스
    - 로그를 확인하는 것이 목표
    - 보통 1시간 단위로 서버에서 초기화, 어디로 옮기냐? CloudWatch로 옮김.
    - 이상 탐지, 모니터링, CPU, Memory, Storage, Network 사용량
      - CPUUtilization

- 애플리케이션 배포 및 데이터 서비스
  - ECR: 컨테이너 이미지 저장소 서비스
    - 컨테이너 이미지 올리는 곳
    - 도커가 절대 강자이지만, 컨테이너의 개념이 표준화가 됨(OCI). 그래서 도커 이미지가 아니라, 컨테이너 이미지라 함.
    - 도커 허브 사용. 회사에서 Private하게 사용하고 싶으면,
  - RDS: 관계형 데이터베이스를 손쉽게 생성, 운영, 확장할 수 있도록 제공하는 완전관리형 서비스
    - MySQL -> Maria
    - Postgres
    - Backend
    - MongoDB, Redis, VectorDB

- 트래픽 분산과 콘텐츠 전송
  - ELB: 여러 대의 서버로 들어오는 네트워크 트래픽을 자동으로 분산해주는 서비스
  - CF(CloudFront): CDN을 통해 사용자에게 콘텐츠를 빠르고 안전하게 전달하는 서비스
    - CDN이라고 부름. 개발과 직접 관련있지는 않음.
    - 업계의 상식, 나의 서버가 어딘가에 있다. 지리적, 물리적으로 사용자와 거리가 있으면, 당연히 느리다.
    - 거리가 멀면 latency가 생기고, 요청이 몰리면 서버에 과부하가 걸림.
  - API GW

- 서버리스 및 배포 자동화
  - Lambda
  - CD(CodeDeploy)
  - EKS
