## 2교시

- VPC: 사용자가 AWS 클라우드 내에서 자신만의 논리적으로 격리된 네트워크를 생성하고 관리할 수 있는 서비스. 여러 서버가 존재할 것. 컴퓨터를 묶어주는 네트워크다.
- IAM: 사용자 계정. 권한 필요 시 요청할 것. 비용 승인, 권한 승인이 필요함. 부트캠프에서는 예산 안에서 진행할 것.
- 보안 그룹(Security Group): Inbound, Outbound
- Route 53: 상식, 개인적으로 서비스를 하고 싶을 때 사용하는 방법 중 하나,
  - 도메인 이름 사는 곳
- Elastic Compute Cloud(EC2):
  - (시스템) 메모리 (휘발)
  - 스토리지: SSD (영구 저장 공간)

- 메가존: AWS와 소비회사 사이 중간 회사

- Elastic IP(탄력적인 IP): 동적 클라우드 환경에서 퍼블릭 IP 주소를 유지할 수 있도록 고정 IP 주소 체계
- S3: 내일
- Elastic File System(EFS): AWS에서 제공하는 스토리지를 직접 설정하거나 용량을 관리하지 않아도, 자동으로 확장, 축소되고 운영이 자동화된 네트워크 파일 시스템
  - 마운트(Mount): 외부 저장 장치(volume, storage, disk) 연결하기 위함

### 후기

- 교육/세미나 아쉬웠던 점: 없음
- 교육/세미나 좋았던 점: 배포 예시로 실제 에러 시나리오 확인이 좋았습니다.
- 강사에 대한 피드백: 감사합니다
- 교육/세미나 평점 이유: AWS의 각 서비스가 어떤 것을 의미하는지 이해할 수 있어 좋았고, 과한 내용은 생략되어 좋았습니다.
- 학습한 내용 요약: AWS, 무중단 배포
- 스스로 개선 및 보완하고 싶은 점: AWS 서비스에 대한 추가 학습
- 성과 평점 이유: 학습에 필요한 키워드 개념을 한 줄 기록하고, 정리하면서 이해하는 과정 수행
