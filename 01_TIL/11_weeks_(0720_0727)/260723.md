- API GW: 클라이언트(웹, 모바일 등)와 백엔드 서비스(EC2) 사이에서 API 요청을 관리, 변환, 보호해주는 완전관리형 서비스
  1. 환경(env) 구분

     3. staging stage 스테이징: 최종 리허설
     4. prod 프로드(live)
  2. 로깅
     1. 요청 응답
  3. 인증

- Lambda: 서버를 직접 관리하지 않고 코드만 실행할 수 있는 서버리스 컴퓨팅 서비스
  - 보통 서버리스(serverless) 컴퓨팅
    - 하드웨어(네트워크): 온프레미스(on-premise)

## 배포

- 위챌 1
  - 빅뱅 배포
    1. 서버 확보
       1. EC2 인스턴스 생성
          1. SSH 22
          2. pem
    2. 서버에 접속
       1. git clone 프로젝트\_URL
       2. uv 설치
       3. uv sync (파이썬 설치, 가상 환경 생성, 의존성 설치)
       4. source .venv/bin/activate
       5. .env 설정
       6. 서버 시동

  - 도커 배포
    1. 도커 이미지 생성
       1. 도커 파일 작성
       2. docker build --platform linux/amd64 -t rag-name .
    2. 만든 도커 이미지를 도커 컴포즈를 실행
       1. docker-compose.yml 작성

- 위챌 2
  - 내가 만든 도커 이미지를 도커 허브에 올린다.
    - generate new token
    - access permissions를 반드시 write가 가능하게 변경
  - EC2에서 해당 이미지를 받아 실행한다.
    - docker compose up
    - 브라우저에서 테스트하기
