## WebRTC

- 초저지연 실시간 통신

## HLS

- 어도비 플래시 없이 영상을 송출하는 Google의 기술
- 온디멘드 방식: 영상이 이미 완성될 때 가능
- 월드컵도 온디멘드를 매번 실행하여 실시간 통신 시도
- BTS 윔블던 공연 V
- 유튜브 지연(latency)이 10초
- 트위치 latency가 4초
- 초저지연을 2초가 목표

### 무엇을 공부?

- 사용하는 언어에 익숙해지기
- 빈출 주제 먼저 공부하기
- 어려운 문제 대비하기

- 사용하는 언어 익숙해지기
  - 코드업: 기초 100제
  - 프로그래머스: 코딩 기초 트레이닝
  - 정올: 단계별 문제
  - 구름 내부 플랫폼

- 빈출 주제 먼저 공부하기
  1. 구현, 시뮬레이션
  2. 완전 탐색, 그래프(BFS, DFS, Dijkstra)
  3. 큐, 스택, 덱, 그리디, DP, 집합과 맵, 파싱, 트리
  4. 이분 탐색, 누적합, 슬라이딩 윈도우, 투포인터, 분리 집합

- 어려운 문제 대비하기
  - 최단 경로(벨먼 포드, 플로이드 워셜, ...)
  - 최소 신장 트리
  - 위상 정렬
  - SCC
  - DP - 비트마스킹, LIS, LCS, ...
  - 정수론(에라스토테네스의 체, 유클리드 호제법, 모듈러 역원)
  - 그 외 세그먼트 트리, LCA, Sparse Table, ...

### 어떻게 공부?

- 시간복잡도 (공간복잡도) 계산하기
- 같이 공부하기
- 효율적으로 공부하기

#### 시간 복잡도

- 알고리즘의 성능 평가 지표로 이용
- 표기할 때는 "점근 표기법"을 이용
- 그 중 Big O notation을 가장 많이 사용

#### 같이 공부하기

- 커뮤니티(solved.ac), 주변인을 적극 활용하기
  - 많은 정보를 접할 수 있음
  - 새로운 관점을 배우기도 함
  - 비슷한 신력의 사람과 함께 공부하는 것도
- LLM을 이용해도 좋음.
  - Opus 4.7 등 정도의 모델에서 정확한 답변을 제공함. (코테의 난이도)

#### 효율적으로 공부하기

- 시간은 한정된 자원
  - 풀이 시간을 정해두고, 시간을 넘기면 풀이를 보는 것이 좋다.
  - 고민하는 시간도 중요하지만, 많은 문제를 푸는 것이 중요
- 다양한 자료로 공부 (IOI Korea)
  - 강의, 블로그, 책 등 특히 인터넷에 많은 자료
- 틀린 문제 Upsolve(오답)은 꼭 하기
  - 풀이를 보고 넘기지 말고 꼭 직접 구현하기
  - 새로운 문제를 내 것으로 만드는 과정이 필요
  - 다른 사람의 코드도 참고하면 좋음.
- 쉬운 문제를 빠르고 정확하게 푸는 연습
  - 이전에 봤던 문제를 다시 보더라도 복붙
  - 구현 속도를 높이는 데 도움
- 랜덤 디펜스
  - 시간 제한을 두고, 적절한 난이도 범위에서 랜덤으로 문제를 뽑아 풀기
  - 난이도, 정답 알고리즘은 가린 채로 진행
- 틀렸을 경우
  - 직접 케이스를 만들어서 넣어보기
  - 극단적인 경우를 생각해보기 (Edge case 찾기)

### 어떻게 푸나?

- 문제 환원하기
  - 풀이에 필요없는 내용을 걷어내고 문제를 간단하게 요약하기
  - 요약된 문제를 이미 아는 문제로 환원하기
- 감이 안 온다면
  - 작은 N에 대해 직접 답을 구하기
  - N 제한을 보고 알고리즘을 유추해보기
  - 큰 문제를 작은 문제로 쪼개서 해결해보기
  - Subtask를 해결하고 부분점수를 받기

#### 구체화 잘 하기

- 발상 -> 구현
  - 구현 중 설계 수정
  - 키보드를 잡고 고민하는 시간 증가
  - 디버깅 시간 증가
- 발상 -> 구체화 -> 구현 ✅ (수도코드 수준, 시간복잡도)
  - 코드 작성 속도 향상
  - 구현 실수 감소
  - 디버깅 용이
- 수도 코드를 작성 가능?
- 시간 복잡도가 맞는지?
- 배열에서 인덱스의 의미가 일관되는가?
- 구현에 필요한 자료구조와 변수의 역할이 명확한가?
- dp 초기값이 잘 정의되었는가, 상태 정의는 올바른지, 전의가 올바른지
