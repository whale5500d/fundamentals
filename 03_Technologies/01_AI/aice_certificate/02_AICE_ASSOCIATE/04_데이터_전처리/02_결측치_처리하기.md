# 데이터 전처리

## 결측치 처리하기

- 결측치
  - 데이터에 값이 없는 것
  - N/A: Not Available, does not exist, missing
  - Null: empty(null) object
  - NaN: Not a Number(python)
- 0은 결측치일까? No
- 결측치는 데이터를 분석하는 데 있어 반드시 처리해야 하는 값이다.
- 좋은 성능의 AI 모델을 만드는 데 중요한 요소이며, 추가적인 조사나 통상적인 방법으로 결측치를 처리해야 한다.

결측치 처리의 특징

1. 실무자의 견해가 많이 반영되는 단계
   - 같은 데이터도 실무자 의견에 따라 결과가 달라질 수도 있고, 임의대로 결측치 처리 시 분석 결과가 매우 틀어질 수 있음.
2. 시간이 많이 투자되어야 함.
   - 데이터 현실을 반영한 결측치 처리가 되어야 데이터 분석을 정확하게 진행할 수 있음.
3. 가장 쉬운 결측치 제거 방법은 결측치를 제거하는 것
   - 하지만, 막대한 데이터 손실 동반
4. 결측치를 단순 대체할 경우
   - 데이터에서 편향(bias)이 발생할 수 있음

결측치 처리 방법

1. 제거하기
   1. Listwise 방식: 결측치가 존재하는 전체 행 삭제
      - 최대치의 정보손실 발생
   2. Pairwise 방식: 모든 변수가 결측치인 경우 해당 행 삭제
      - 정확한 데이터 분석이나 최적 모델 구현 시, 나쁜 영향을 줌
   - 데이터의 손실 많이 발생
2. 채우기
   1. 대체하기: 결측치를 평균화 값으로 대체
      - 가장 널리 사용되는 평균화 값은 mean(평균), median(중앙값), mode(최빈값)
        - 범주형 데이터일 경우, 최빈값 활용
        - 수치형 데이터일 경우, 평균, 중앙값 활용
   2. 예측하기: 결측치를 상관관계 등으로 예측하여 대체
      - 결측치의 특성이 무작위로 관찰되는 것이 아니라고 가정
      - 상관관계나 예측모델을 사용하여 채움
   - 데이터의 손실을 발생하지 않으나, 모든 결측치가 동일한 값을 가질 수 있음. (편향 발생 가능성)
   - 채워진 값에 의해 전체 평균, 중앙값, 상관관계 등이 영향을 받을 수 있음.
3. 실제 함수
   1. `fillna('standard')`: 결측치를 지정한 값으로 채우는 메서드
   2. ⚠️ `interpolate()`: 결측치를 같은 간격으로 채우는 메서드
      - 예시: 360, 16 사이 N/A가 있을 경우, 두 수의 중앙값인 188을 채움
      - 수치형 데이터를 채움
   3. `dropna()`: 결측치가 있는 레코드를 제거하는 listwise 방식(`dropna(how='all')`는 pairwise 방식)

## Key Point

1. 결측치 정의
   - Missing Value로 데이터에 값이 없는 것을 의미
   - 결측치를 N/A, Null, NaN로 표기
2. 결측치 처리
   - 결측치 제거하기: 한 행 모두 결측치일 때, 컬럼에 많은 결측치 포함
   - 결측치 채우기: 연속형 -> 평균값/중간값, 범주형 -> 최빈값
