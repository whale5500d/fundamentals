# 머신러닝 개념과 기술 원리

## 머신러닝 기본 개념

- 머신러닝 프로세스
  1. 데이터 수집
  2. 데이터 정리 (테스트 데이터셋 준비)
  3. 모델 학습
  4. 모델 테스트 (테스트 결과로 모델 재학습)
  5. 모델 배포

- 과거 vs AI/머신러닝
  - 과거
    - 사람이 직접 데이터 패턴을 찾아 알고리즘 코딩하여 결과를 얻음
    - Model
  - AI/머신러닝
    - 데이터와 결과를 기반으로 스스로 패턴 학습하고 이를 이용해서 예측

## 머신러닝 모델

1. 선형 회귀 (Linear Regression)
   - 가설: 공부를 많이 하면 공부를 잘 할 것이다.
   - 예시: 학습시간과 토익점수
   - 목표: 표시한 점들을 가장 잘 표현하는 하나의 직선 파악하기
   - 가장 잘 맞는 직선을 구한다는 것의 의미
     - 기울기(가중치, w)가 얼마나 정확한지
     - 컴퓨터는 어떤 w가 최적인지 알기 위해 "**직선별 Cost Function의 최소값**"을 사용한다.

2. Cost(Error, Loss) Function
   - (실제값 - 예측값)^2 / N = MSE(Mean Squared Error)
   - 경사 하강 알고리즘(Gradient Descent Algorithm)
   - 그래프 상 기울기가 0인 지점이 비용이 가장 낮다.

## 머신러닝 기술 원리

1. 머신러닝을 위해 필요한 것
   - 데이터 확보
   - 잘 정리된 데이터 확보가 중요

2. 지도학습 vs 비지도학습
   - 지도학습(Supervised Learning)
     - 정답을 알려주면서 진행하는 학습
     - 데이터와 레이블(정답)이 함께 제공됨
     - 레이블(Label) = 정답, 실제값, 타깃, 클래스, y 등
   - 비지도학습(Un-Supervised Learning)
     - 정답없이 진행되는 학습
     - 데이터 자체에서 패턴을 찾아내야 할 때 사용

3. 지도학습 모델 종류
   - 분류모델(Classification)
     - 레이블의 값들이 이산적으로 나눠질 수 있는 문제에 사용
     - 예시: 남녀 분류, 학점 분류
   - 회귀모델(Regression)
     - 레이블의 값들이 연속적인 문제에 사용
     - 예시: Tip 값 예측, 삼성전자 주가 예측

4. 지도학습 데이터셋 구조
   - 각 열(column)을 특징/속성(feature)라고 함
   - 각 행(row)을 예제(Example) 데이터라고 함
   - 데이터 열 중에 하나를 선택해서 레이블로 사용함

5. 데이터셋 분리
   - Train, Validation, Test로 분리

6. 모델 선택 및 성능 평가
   - 적합, 과적합, 과소적합
     - 과적합: 학습 데이터는 제대로 학습하지만, 평가 데이터는 결과가 안 좋은 경우
     - 과소적합: 더 많은 학습 필요
   - Epochs은 학습 횟수
   - Epochs가 늘어날수록 Error 감소
   - Epochs가 늘어날수록 Error가 다시 늘어나면 과적합
   - 성능 지표
     - 학습이 끝난 후 모델을 성능 평가하는 용도로 사용됨
     - 회귀 모델 성능 지표: **MSE, MAE, R2**
       - MSE는 오차 제곱의 평균, MAE는 오차 절대값의 평균
       - MAE보다 MSE가 학습에 더 큰 영향을 미침(제곱이므로)
       - R2는 회귀 모델에서 독립변수가 종속변수를 얼마나 잘 설명해주는지 나타내는 지표
       - R2 Score가 1에 가까울수록 좋은 모델
     - 분류 모델 성능 지표: **정확도, 정밀도, 재현율, F1-점수**

7. 회귀 성능 지표
   1. MSE(Mean Squared Error): 예측값에 대한 실제값의 오차를 제곱한 값의 평균
   2. MAE(Mean Absolute Error): 예측값에 대한 실제값의 오차를 절대값의 평균
      - 이상치 데이터가 있을 때, MAE보다 MSE가 더 큰 영향을 받으므로 주의해서 사용해야 함.
   3. R2, 결정계수: 회귀 모델에서 독립 변수가 종속 변수를 얼마나 잘 설명하는지 나타내는 지표
      - R2 Score가 1에 가까울수록 좋은 모델
      - R2 = 1 - ((실제-예측)^2의 합) / ((평균-예측)^2의 합)

8. 오차 행렬(Confustion Matrix)
   - 분류 모델의 성능을 오차행렬을 기반으로 수치로 표현
   - 첫 번째 자리: 맞느냐 틀렸느냐를 의미
   - 두 번째 자리: 예측한 값을 의미
     - TP: True로 예상했고, 맞았다.
     - TN: False로 예상했고, 맞았다.
     - FP: True로 예상했고, 틀렸다.
     - FN: False로 예상했고, 틀렸다.

9. 분류 성능 지표
   1. 정확도(Accuracy) = (TP+TN) / (TP+FN+FP+TN)
      - 암환자 1명에 건강인 99명을 대상으로 테스트를 하면,
      - 음성으로 진단했을 때 정확도는 99% (정확도의 함정)
   2. 정밀도(Precision) = (TP) / (TP+FP)
      - True로 예측한 것 중, 실제로 True인 것의 비율
      - 날씨 예측 모델이 맑다고 예측했을 때, 실제 날씨가 맑은 비율
      - 스팸 메일이라고 예측한 것 중에서 실제 스팸인 비율
   3. 재현율(Recall) = (TP) / (TP+FN)
      - 실제 True인 것 중, True라고 예측한 것의 비율
      - 실제 날씨가 맑은 날 중에서 모델이 맑다고 예측한 비율
      - 실제 암환자들 중에서 암환자라고 예측한 비율
   4. F1 점수 = 2 \* ( P \* R / (P + R))
      - 정밀도와 재현율의 조화평균
